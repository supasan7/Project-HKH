# Requirements: Homestay Management System

## 1. Problem Statement
ปัจจุบันธุรกิจโฮมสเตย์ของครอบครัวยังใช้การจดบันทึกด้วยมือ ทำให้เกิดปัญหา:
- ข้อมูลรายรับ-รายจ่ายไม่เรียลไทม์ ตรวจสอบย้อนหลังยาก
- ความเสี่ยงในการจองซ้อน (Double Booking)
- ข้อมูลขาดความโปร่งใสและการเข้าถึงจากหลายที่พร้อมกัน

## 2. User Roles
1. **Staff (พนักงาน):** ดูสถานะห้อง, เพิ่มการจอง, เช็คอิน-เช็คเอาท์
2. **Owner (เจ้าของ):** ทำได้เหมือน Staff และสามารถดูรายงานการเงิน, จัดการข้อมูลห้อง, แก้ไขรายจ่ายได้

## 3. Functional Requirements
### ระบบจัดการการจอง (Booking Management)
- [ ] พนักงานสามารถตรวจสอบห้องว่างตามวันที่ระบุได้
- [ ] ระบบต้องคำนวณราคาสุทธิโดยอัตโนมัติ
- [ ] สามารถบันทึกสถานะการชำระเงินได้ (มัดจำ/จ่ายเต็ม)

### ระบบจัดการการเงิน (Financial Tracking)
- [ ] เจ้าของสามารถบันทึก "รายจ่าย" แยกตามหมวดหมู่ (ค่าไฟ, ค่าแม่บ้าน)
- [ ] ระบบต้องเชื่อมโยงรายได้จากการจองเข้าสู่ระบบบัญชีโดยอัตโนมัติ
- [ ] แสดงสรุปกำไร-ขาดทุน รายเดือน

## 4. Non-Functional Requirements
- **Reliability:** ระบบต้องจัดการ Transaction ของ Database ได้ถูกต้อง (ถ้าจองไม่สำเร็จ ต้องไม่มีข้อมูลค้างในระบบ)
- **Security:** มีระบบ Login และแบ่งสิทธิ์ (Role-based Access Control)
